1、Hadoop生态圈

1.1大数据技术概览

大数据是指超出传统数据库工具收集、存储、管理和分析能力的数据集。与此同时，及时采集、存储、聚合、管理数据，以及对数据深度分析的新技术和新能力，正在快速增长，就像预测计算芯片增长速度的摩尔定律一样。大数据的基本特征如下：

大数据的应用场景：一是基于大数据的数据仓库，他可以处理新的数据类型（包括非结构化数据和半结构化数据）且具有良好的扩展性、可用性和廉价性；二是基于大数据的实时流处理。

大数据技术编年史大致分三个阶段：前Hadoop时代，Hadoop时代，后Hadoop时代。

第一个阶段是2002年到2006年（即2006年之前），称之为前Hadoop时代。前Hadoop时代论证并实现了大数据实现的理论基础（HDFS和MapReduce）。前Hadoop时代的里程碑（MileStone）式的事件是Google发表的两篇论文：GFS和MapReduce论文，但是Google并没有开放源代码。Doug Cutting用Java实现了这两篇论文。一个解决了海量数据存储的问题，另一个解决了海量数据并行计算的问题。
第二个阶段是2007年到2012年，称之为Hadoop时代。这几年是Hadoop高速发展的几年，也是Hadoop从科研走向商业应用的关键几年。Hadoop时代的里程碑Hadoop正式成为Apache的顶级项目，成立了第一家从事Hadoop商业化的公司Cloudera。同时Hadoop及其生态圈初步构建完毕。比如Hadoop上第一个SQL框架Hive，HDFS的NameNode支持HA等。

第三个阶段是2013年以后，称之为后Hadoop时代。后Hadoop时代继续完善Hadoop的生态圈，同时处理数据的功能和效率上有大幅提高。里程碑事件是Spark取代MapReduce成为Apache Hadoop默认执行引擎。

1.2分布式文件系统HDFS

HDFS是Hadoop分布式文件系统（Hadoop Distributed File System）的缩写。，HDFS是GFS的开源实现，也是Apache Hadoop的核心子项目，在开源大数据技术体系中，地位无可替代。

HDFS具有高容错、高可用、高扩展、支持海量数据存储、构建成本低、安全可靠、适合大规模离线批处理等优点。其缺点是：不适合低延迟数据访问，不适合大量小文件存储，不支持并发写入，不支持文件随机修改，仅支持追加写入。

2、 大数据平台

2.1分布式计算框架

分布式计算框架主要讲MapReduce和Spark。

MapReduce是面向批处理的分布式计算框架，一种编程模型：MapReduce程序被分为Map（映射）阶段和Reduce（化简）阶段。其设计初衷是解决搜索引擎中大规模网页数据的并行处理。其核心思想是分而治之，并行计算，移动计算而非移动数据。

其特点是：计算跟着数据走，具有良好的扩展性：计算能力随着节点数增加，近似线性递增，高容错性，状态监控，适合海量数据的离线批处理，降低了分布式编程的门槛。

MapReduce的适用场景：数据统计，搜索引擎构建索引，海量数据查询，进行复杂数据分析算法实现。但不适用于OLAP、流计算、DAG计算。

Spark由加州大学伯克利分校的AMP实验室开源，是一种采用Scala语言开发的大规模分布式通用计算引擎。具有高吞吐、低延时、通用易扩展、高容错等特点。

2.2分布式消息队列Kafka

Kafka是基于发布/订阅的分布式消息系统，由Linkedin开发，用Scala语言编写。具有消息持久化、高吞吐、高容错、易扩展和同时支持离线、实时数据处理等特点。其工作机制如下图：

2.3可视化大数据ETL工具Transporter

Transporter一种可视化的大数据ETL（Extract-Transform-Load）工具

是构建数据仓库的重要一环，负责完成数据从来源端经过抽取（extract）、交互转换（transform）、加载（load）至目的端的过程。支持从不同数据源获取数据，对数据的复杂转换操作，并最终将数据落地成不同格式。

其技术特点：支持丰富的数据源，可直接将Oracle/DB2等传统DBMS数据导入至TDH，同Sqoop相比，在不失效率的情况下将操作化繁为简；支持丰富的导出格式，可直接导出数据至传统关系数据库、ElasticSearch，以及实现跨集群导出；支持多种数据转换操作，提供多种常见的数据转换操作，帮助实现数据的清洗和加工。

2.6 TDH集群的角色分配

非实时集群的角色分配的总体原则：集群需分配单独的管理节点；集群规模较大时，需多个机柜（至少两个）；高可用的管理角色尽量分散到多个机柜中；各管理节点上的服务角色尽量保证负载均衡。

实时集群的角色分配的总体原则：因为实时集群对实时性要求很高，为避免非实时集群对它的影响（主要CPU和磁盘IO），建议使用单独的几个节点做实时集群。

3、服务安装与使用

3.1分布式SQL引擎Inceptor

Transwarp Inceptor是星环科技推出的用于数据仓库和交互式分析的大数据平台软件，它基于Hadoop和Spark技术平台打造，加上自主开发的创新功能组件，有效的解决了企业级大数据数据处理和分析的各种技术难题，帮助企业快速的构建和推广数据业务。

其特点为：Hadoop领域对SQL支持最完善；唯一支持完整分布式事务处理；具有优异的大数据处理和分析性能；提供便捷的SQL、PL/SQL开发调试辅助工具WaterDrop。

3.2 SQL集成开发环境Waterdrop

Waterdrop是为开发人员和数据库管理人员提供的数据库管理工具，它可进行跨平台管理，可作为Inceptor SQL客户端，除了Inceptor还支持并兼容其余多种数据库。它具有有四个主要的功能模块：Database Navigator、SQL Editor、SQL Executor、Data Viewer/Editor，分别用来帮助用户实现数据库管理、SQL编辑、SQL执行、数据操作这四项功能。此外，Waterdrop还提供了一系列数据库开发和ETL工具,例如存储过程测试器, 数据导出工具(支持excel、csv、html、xml、json），同源/异源数据库间数据传输等等。

3.3 实时流处理引擎Slipstream

批处理：批量处理、调度延时、处理延时

流处理：流式处理、低延时

Slipstream是融合事件驱动与微批处理的实时分布式流计算引擎。其优势：微批模式和事件驱动模式的一体化、极高的易用性、在一些条件下，采用Slipstream的方式甚至比编程方式获得更高的性能提升、产品化程度高，迁移成本低。

4、集群运维

4.1分布式NewSQL数据库Hyperbase

Transwarp Hyperbase实时数据库是建立在Apache HBase和Elasticsearch基础之上，融合了多种索引技术、分布式事务处理、全文实时搜索、图形数据库在内的实时NoSQL数据库。Hyperbase可以高效地支持企业的在线OLTP应用、高并发OLAP应用、批处理应用、全文搜索或高并发图形数据库检索应用，结合Inceptor高速SQL引擎，是企业创建可扩展在线运营数据库（Operational Database）或者实时分析型数据库(ODS - Operational Data Store)的最佳选择。其架构图如下：

4.2分布式全文搜索引擎Seaarch

Transwarp Search基于开源的Elasticsearch进行开发，在其基础架构的基础上进行了多种功能优化，并通过Esdrive实现了SQL交互方式。Transwarp Search是一个可扩展的分布式全文搜索和分析引擎。其特点是：分布式实时文档存储；分布式实时搜索分析；SQL引擎与搜索引擎相融合；具有高扩展性。在TDH平台中，Search扮演两种角色：一是作为Hyperbase全文检索的底层实现；二是作为一个独立的服务，既是一个分布式文件存储系统，又是一个强大的全文搜索引擎。

4.3平台运维管理Transwarp Manager

Manager是集群图形化管理应用程序。通过Manager可视性来控制 Hadoop 集群，您可以轻松地部署、安装、监控和集中操作整个的 TDH集群。如下所示，Manager承载管理控制台、Web 服务器和应用程序逻辑。它负责安装软件、配置、启动和停止服务，以及管理在集群运行的服务。

Manager由以下几部分组成:

集群代理：在集群中的每台主机上都必须安装代理。它负责启动和停止进程，安装、卸载配置、 报警以及监视主机。（TDH安装时候会自动安装）

数据库：存储系统的配置和监视日志信息。通常有多个逻辑数据库运行在一个或多个数据库服务器。例如，TDH可以监控守护进程使用不同的逻辑数据库。

客户端：Manager会提供一个接口，用于客户端与服务器进行交互。管理控制台是基于 Web 的用户界面，管理员通过用户界面管理群集。
